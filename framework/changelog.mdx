<Update label="v1.0.8" description="March 14, 2026">
### Improvements
- feat: Overhaul theme engine with `$overrides` and `$dark-overrides` Sass maps for foundational, granular token control.
- feat: Introduce 4-Stage Build sequence (build:css -> build:snap -> build:zip -> build:customer) for lean, professional handouts.
- feat: Integrate a comprehensive Theme & Typography Preset switcher into the `project-template` scaffold for rapid prototyping.
- feat: Modularize offcanvas system into a standalone `offcanvas.php` template part, decoupled from the header for greater flexibility.
- feat: Implement dark mode FOUC (Flash of Unstyled Content) prevention script in `base.php` to ensure seamless loading.
- feat: Reimplement theme management with separate Vite-compiled SCSS entry points for faster builds and cleaner isolation.
- feat: Implement high-fidelity responsive heading scale (text-h2 -> text-h1 -> text-display-6) across the project scaffold.
- feat: Add high-fidelity header parameters (`$isDark`, `$isFloat`) for professional dynamic layout control.
- feat: Synchronize design tokens across `_config.scss` and `_tokens.scss` for centralized customization.
- feat: Enable professional media optimization and blur defaults in `squeditor.config.js`.
- feat: Add RGB CSS variables for all color tokens to support Tailwind opacity modifiers and update related SCSS utilities.
- feat: Refactor SCSS tokens to explicitly support light mode, extending theme-two styles to light mode.
- feat: Enhance build script to support HTML subfolders and compartmentalized `distSubfolder` mapping.
- feat: Introduce and integrate `cli-ui` utility for standardized and enhanced command-line output across scripts.
- feat: Implement conditional content stripping for dev/demo blocks and enhance font asset handling across build scripts.
- feat: Propagate framework improvements to AI skills (`.agent/skills`), including specialized landing page instructions.

### Refactors & Internal
- refactor: Centralize common utility functions and constants into a new `core.js` module for framework-wide reuse.
- refactor: Improve Prettier execution logic and error reporting in the packaging pipeline.
- refactor: Refine theme switcher and unassigned theme removal regex to handle varied HTML tag formatting.
- feat: Sync documentation with new typography token patterns and advanced `$overrides` map examples.

### Bug Fixes
- fix: Ensure heading font-weight correctly inherits from `--sq-heading-font-weight` token across all themes.
- fix: Refine CLI experience with silent ZIP output and improved build phase headers.
</Update>
